#### Атрибуты вопроса:

1. <b>id вопроса</b> - идентификационный номер вопроса <br><br>Должен быть уникальным в рамках одной сессии одного преподавателя<br><br>Для простоты использования рекомендуем указывать в качестве id вопроса его порядковый номер в таблице (как на примере выше)
<hr>
2. <b>Текст вопроса</b> - текст вопроса, который задается обучающемуся<br><br>В случае если вопрос задается относительно иллюстрации (флаг, формула и т.п.), то ссылка на иллюстрацию указывается отдельно в поле фото
<hr>
3. <b>Тип вопроса</b> - тип вопроса (один верный вариант, несколько верных вариантов, открытый вопрос)
<br><br><b>one_choice</b> - данное значение прописывается в ячейке типа вопроса для вопросов с одним вариантом верного ответа
<br><br><b>multi_choice</b> - данное значение прописывается в ячейке типа вопроса для вопросов с несколькими вариантами верных ответов
<br><br><b>write_answer</b> - данное значение прописывается в ячейке типа вопроса для открытого типа вопросов - ответ вводится с клавиатуры учеником самостоятельно
<br><br><b>ВАЖНО:</b> для повышения эффективности использования открытых вопросов существует функция "Варианты для открытых вопросов" (см. пункт 9)
<hr>
4. <b>Правильный(е) ответ(ы)</b> - вариант правильного ответа
<br><br>Несколько правильных вариантов ответов (тип вопроса multi_choice) указываются в одной ячейке через точку с запятой и один пробел - Москва; Новосибирск; Барнаул
<hr>
5. <b>Неправильные варианты</b> - варианты некорректных ответов
<br><br>Несколько неправильных вариантов ответов (тип вопросов one_choice, multi_choice) указываются в одной ячейке через точку с запятой и один пробел - Берлин; Осло; Прага
<br><br>Не указывается для открытого типа вопроса (write_answer). Ячейка остается пустой
<hr>
6. <b>Тэг (тема) вопроса</b> - тэг вопроса, по которому он относится к конкретному заданию
<br><br>Механика <b>ТЭГов</b> используется для управления заданиями (тестами)
<br><br>За счет тэга(ов) происходит установление соответствия в паре "Тест (задание) - вопрос(ы)"
<br><br>Каждое задание (тест) также содержит <b>ТЭГ</b>
<br><br>При загрузке файла с вопросами сервис <a href="https://eduquiz.ru" target="_blank">EduQuiz</a> создает задания (тесты) на основе логики тэгов. В каждый тест выбираются (включатся) вопросы с совпадающими тэгом(ами)
{{< img name="files/testtag*" title="EduQuiz тестирование через телефоны" class="article-image">}}
<br><br>В приведенном выше примере при загрузке файла на сервис <a href="https://eduquiz.ru" target="_blank">EduQuiz</a> будут созданы 3 теста (задания): "Про одежду", "Про цвета", "Про космос"
<br><br>В каждое из заданий попадут вопросы с указанными тэгами "работа"; "фрукты" + "овощи"; "космос"
<br><br>Использование данной механики управления банком вопросов приобретает ценность при управлении значительным объемом вопросов (100+) на смежные тематики
<br><br>В случае использования незначительного количества вопросов и заданий рекомендуется указывать простые тэги, например - тему занятия ("простые_дроби")
<hr>
7. <b>Баллы за вопрос</b> - количество баллов, которое получает ученик в случае верного ответа
<br><br>Для вариантов с несколькими вариантами правильных ответов (тип multi_choice): в случае, если ученик указал лишь некоторые верные ответы (но не все), либо же указал в т.ч. и неверные ответы - он получает ноль баллов
<hr>
8. <b>Фото вопроса</b> - картинка (иллюстрация) к вопросу
<br><br>В ячейке прописывается ссылка на картинку, размещенную в интернете. Вид ссылки: http://nl-a.ru/img/header-logo.png
<br><br>Иллюстрации могут быть найдены либо через поисковики (яндекс, google), либо загружены самостоятельны на открытые хостинги фотографий (например: альбомы в vk.com)
<br><br>Данный атрибут является опциональным. Если иллюстрации нет - ячейка остается пустой
<hr>
9. <b>Варианты regex для открытых вопросов</b> - синонимы (корни слов) варианта верного ответа
<br><br>Типовая проблема автоматизированной проверки открытых вопросов: обучающиеся часто пишут ответ, который не строго соответствует прописанному правильному ответу в системе, но строго совпадает по смыслу
<br><br>Например вместо правильного ответа "Юрий" тестируемый может написать "Юра". Смысл (семантика) совпадает; синтаксис - нет
<br><br>Для минимизации подобных рисков при автоматизированном использовании открытых вопросов в сервисе <a href="https://eduquiz.ru" target="_blank">EduQuiz</a> предсмотрен функционал "создания синонимов" (регулярные выражения, regex) правильных ответов
<br><br>В данной ячейке прописываются "синонимы" корректного ответа
<br><br>Рекомендуем прописывать лишь корни ответа с указанием специального символа * в конце корня. Несколько вариантов ответов указываются через точку с запятой через пробел
<br>
<blockquote>Пример использования "Регулярного выражения":</blockquote>
<b>Вопрос</b>: "Укажите одну из исторических столицу России"
<br><br><b>Верный ответ</b>: Москва
<br><br><b>Варианты ответов</b>: Москв* ; Ладог* ; Новгород* ; Киев* ; Владимир* ; Санкт* ; Петроград* 
<br><br>Между "звездочкой" и точкой с запятой не должно быть пробела в ячейке; в примере стоит пробел в силу особенностей форматирования текста на веб-сайтах
<br><br><br><br>

### Способ #2: Создание вопросов и заданий через панель администратора на сервисе <a href="https://eduquiz.ru" target="_blank">EduQuiz</a>

Для создания вопроса непосредственно на сервисе <a href="https://eduquiz.ru" target="_blank">EduQuiz</a> необходимо авторизовать (через профиль в одной из социальных сетей)

{{< img name="files/social*" title="EduQuiz тестирование через телефоны" class="article-image">}}
<br>

Затем нажать на кнопку "Create" (создать)

{{< img name="files/create*" title="EduQuiz тестирование через телефоны" class="article-image">}}
<br><br>
Затем в меню (слева вверху) нажать на вкладку "Вопросы"
<br><br><br>
{{< img name="files/qstntab*" title="EduQuiz тестирование через телефоны" class="article-image">}}
<br><br><br>
В разделе управления вопросами для создания нового вопроса - нажать кнопку "Плюс" (справа внизу)
<br><br><br><br>
{{< img name="files/plus*" title="EduQuiz тестирование через телефоны" class="article-image">}}
<br><br>
Заполнить все поля вопроса
<br><br><br>
{{< img name="files/qstncreate*" title="EduQuiz тестирование через телефоны" class="article-image">}}
Нажать кнопку "Сохранить"

Все. Вопрос создан и сохранен на сервисе <a href="https://eduquiz.ru" target="_blank">EduQuiz</a>

<blockquote>ВАЖНО: сервис <a href="https://eduquiz.ru" target="_blank">EduQuiz</a> не хранит вопросы длительное время. Вопросы удаляются через 23 часа после создания, либо в момент окончания сеанса тестирования</blockquote>

Для сохранности создаваемого контента рекомендуется сохранять (выкачивать) с сервиса файлы с вопросами. Для этого во вкладке "Вопросы" необходимо нажать на иконку "скачать файл с вопросами"
